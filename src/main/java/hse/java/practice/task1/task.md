Необходимо реализовать методы поворота кубика Рубика.

Tag: cube

Кубик представляет собой классический куб 3×3×3:
- 6 граней;
- каждая грань состоит из 9 элементов (стикеров);
- каждый элемент однозначно определяется своим цветом.

Каждую из граней можно вращать по часовой стрелке или против часовой стрелки.
При выполнении поворота:
- изменяется расположение элементов на самой вращаемой грани;
- переставляются элементы на четырёх соседних гранях, примыкающих к этой грани.

В рамках задачи необходимо:
- формально представить состояние кубика (структуры `Cube`, `Edge`, `CubeColor`, `EdgePosition` уже заданы или должны быть использованы согласно условию);
- реализовать операции поворота граней во всех допустимых направлениях согласно перечислению `RotateDirection`;
- гарантировать корректное изменение состояния кубика при каждом повороте, чтобы последовательность поворотов давала ожидаемый результат (см. тесты в `CubeSimpleTest`).

Повороты, смотрим на фронтальную грань, от нее считаем все повороты. 

Если смотреть фронтально на каждую грань, левый верхний угол будет иметь индексы 0, 0
Напрвление поворота для грани определяется как будто мы смотрим на нее фронтально
