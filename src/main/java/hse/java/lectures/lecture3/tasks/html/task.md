# Валидация HTML-документа

Tag: html

## Условие

Реализовать класс `HtmlDocument`, который читает HTML-текст из файла и проверяет корректность структуры. Если документ невалиден, при создании объекта выбрасывается исключение.

## Входные данные

Путь к файлу с HTML-текстом. Документ может содержать только поддерживаемые теги и произвольный текст между тегами.

## Поддерживаемые теги

Разрешены только следующие теги (регистр не важен): `html`, `head`, `body`, `div`, `p`.

- Открывающий тег: `<tagName>` или `<tagName ...>` (атрибуты игнорируем).
- Закрывающий тег: `</tagName>`.
- Одиночные теги не используются — все теги парные.

## Правила валидности

1. **Корректная вложенность**  
   Каждый закрывающий тег должен соответствовать последнему открытому и ещё не закрытому тегу.  
   Пример невалидного фрагмента: `<div><p></div></p>` — закрытие не по порядку.

1. **Все теги закрыты**  
   У каждого открывающего тега есть соответствующий закрывающий.  
   Пример невалидного: `<div><p>text` — тег `p` (и при необходимости `div`) не закрыт.

1. **Нет лишних закрывающих тегов**  
   Закрывающий тег не должен появляться до открывающего или после того, как соответствующая пара уже «закрыта».  
   Пример невалидного: `</div>` в начале документа или `<div></div></div>`.

1. **Корректность имени тега**  
   В документе встречаются только поддерживаемые теги. Любой другой тег (например, `<span>`, `<script>`) делает документ невалидным.

1. **Структура документа**  
   Ровно один корневой тег `html`, внутри — не более одного `head` и одного `body`, причём `head` должен идти перед `body`.

## Требования к реализации

- Документ создаётся из файла.
- При невалидном тексте в файле, при создании объекта `HtmlDocument` нужно бросать исключение.
- Не используем внешние библиотеки.
- Разработать систему классов исключений.

## Исключения

1. `UnsupportedTagException` — встретился тег вне списка разрешённых.
1. `UnexpectedClosingTagException` — закрывающий тег без соответствующего открывающего.
1. `MismatchedClosingTagException` — закрывается не тот тег.
1. `UnclosedTagException` — в конце остались незакрытые теги.
1. `InvalidStructureException` — нарушены правила `html/head/body`.

## Примеры

**Валидный документ:**
```html
<html><head></head><body><div><p>Hello</p></div></body></html>
```

**Невалидный документ (нарушена вложенность):**
```html
<html><body><div><p></div></p></body></html>
```

**Невалидный документ (незакрытый тег):**
```html
<html><body><p>text</body></html>
```
